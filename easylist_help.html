Parce que EasyList est un nouveau module (un fork de ListIt2), il n'y a pas de procédure automatique de mise à jour. Vous pouvez quand même suivre cette procédure de déplacement des données.

ATTENTION : cette procédure s'adresse à des techniciens et est à faire avec précaution dans un environnement de test, après avoir sauvegardé vos base de données !!!!

  1. Une fois EasyList installé, créez les instances à l'indentique de celles de ListIt2.
  2. Récupérez les données seulement des tables ListIt2 'category', 'fielddef' et 'fielddef_opts' au format SQL.
  3. Modifiez dans le fichier SQL les noms des entités en remplaçant 'listit' par 'easylist'.
  4. Importez ce fichier dans votre base de données où est installé Easylist.
  5. Vérifiez dans votre site que votre instance Easylist contient bien les catégories et les définitions de champ.
  6. Sur le site Listit2, depuis votre instance, exportez vos articles (en selectionnant tous les champs).
  7. Sur le site EasyList, importez le fichier csv généré et vérifiez que vos articles sont revenus.
  8. Effectuez les réglages dans l'onglet Options.
  9. Enjoy! et offrez-moi un café
